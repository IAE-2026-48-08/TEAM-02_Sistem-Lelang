# PROMPT 1

Saya sedang membangun "Service Katalog Barang" untuk Sistem Lelang menggunakan Laravel. Service ini harus mematuhi standar kontrak integrasi berikut:

1. Format: JSON, UTF-8, Content-Type: application/json.
2. Standar Wrapper Sukses: `{"status": "success", "message": "...", "data": {...}, "meta": {"service_name": "Katalog-Service", "api_version": "v1"}}`.
3. Standar Wrapper Gagal: `{"status": "error", "message": "...", "errors": null}`.
Bantu saya membuat `ItemController` yang mengimplementasikan 3 endpoint minimum viable API:

- `GET /api/v1/items` (Mengambil semua daftar barang lelang)
- `GET /api/v1/items/{id}` (Mengambil detail spesifik 1 barang)
- `POST /api/v1/items/filter` (Memfilter daftar barang berdasarkan kriteria tertentu)
- `POST /api/v1/items` (Menambahkan item baru ke katalog lelang)

Tolong berikan kode controller dan penjelasan struktur responsnya agar seragam.

`app/Helpers/ApiResponse.php`

```
<?php

namespace App\Helpers;

class ApiResponse
{
    public static function success($data, $message = 'Data retrieved successfully', $code = 200)
    {
        return response()->json([
            'status'  => 'success',
            'message' => $message,
            'data'    => $data,
            'meta'    => [
                'service_name' => 'Katalog-Service',
                'api_version'  => 'v1',
            ],
        ], $code);
    }

    public static function error($message = 'Something went wrong', $code = 400, $errors = null)
    {
        return response()->json([
            'status'  => 'error',
            'message' => $message,
            'errors'  => $errors,
        ], $code);
    }
}
```

---

`app/Http/Controllers/ItemController.php`

```
<?php

namespace App\Http\Controllers;

use App\Helpers\ApiResponse;
use App\Models\Item;
use Illuminate\Http\Request;

class ItemController extends Controller
{
    // GET /api/v1/items
    public function index()
    {
        $items = Item::all();
        return ApiResponse::success($items);
    }

    // GET /api/v1/items/{id}
    public function show($id)
    {
        $item = Item::find($id);

        if (!$item) {
            return ApiResponse::error('Item tidak ditemukan.', 404);
        }

        return ApiResponse::success($item);
    }

    // POST /api/v1/items/filter
    public function filter(Request $request)
    {
        $query = Item::query();

        if ($request->has('auction_status')) {
            $query->where('auction_status', $request->auction_status);
        }

        if ($request->has('min_price')) {
            $query->where('starting_price', '>=', $request->min_price);
        }

        if ($request->has('max_price')) {
            $query->where('starting_price', '<=', $request->max_price);
        }

        if ($request->has('keyword')) {
            $query->where('name', 'like', '%' . $request->keyword . '%');
        }

        $items = $query->get();

        if ($items->isEmpty()) {
            return ApiResponse::error('Tidak ada item yang sesuai dengan filter.', 404);
        }

        return ApiResponse::success($items, 'Data filtered successfully');
    }

    // POST /api/v1/items
    public function store(Request $request)
    {
        $validator = validator($request->all(), [
            'name'             => 'required|string|max:255',
            'description'      => 'required|string',
            'starting_price'   => 'required|integer|min:0',
            'auction_deadline' => 'required|date',
            'image_url'        => 'nullable|url',
        ]);

        if ($validator->fails()) {
            return ApiResponse::error('Validasi gagal.', 422, $validator->errors());
        }

        $item = Item::create([
            'name'                => $request->name,
            'description'         => $request->description,
            'starting_price'      => $request->starting_price,
            'current_highest_bid' => 0,
            'auction_status'      => 'OPEN',
            'auction_deadline'    => $request->auction_deadline,
            'image_url'           => $request->image_url,
        ]);

        return ApiResponse::success($item, 'Item berhasil ditambahkan.', 201);
    }
}
```

# PROMPT 2

Berdasarkan kontrak integrasi, setiap endpoint harus diproteksi menggunakan API Key pada Request Header dengan nama key X-IAE-KEY dan value-nya adalah NIM saya.

Tolong buatkan Middleware di Laravel untuk memvalidasi header tersebut secara global atau spesifik pada route API v1.

---

# AI RESPONSE

Berikut implementasi Middleware untuk validasi `X-IAE-KEY`:

---

`app/Http/Middleware/ApiKeyMiddleware.php`

```
<?php

namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;

class ApiKeyMiddleware
{
    public function handle(Request $request, Closure $next)
    {
        $apiKey = $request->header('X-IAE-KEY');

        if (!$apiKey || $apiKey !== env('IAE_API_KEY')) {
            return response()->json([
                'status'  => 'error',
                'message' => 'Unauthorized. API Key tidak valid atau tidak ditemukan.',
                'errors'  => null,
            ], 401);
        }

        return $next($request);
    }
}
```

---

Daftarkan di `bootstrap/app.php`

```
->withMiddleware(function (Middleware $middleware) {
    $middleware->alias([
        'api.key' => \App\Http\Middleware\ApiKeyMiddleware::class,
    ]);
})
```

---

Daftarkan di `routes/api.php`

```
<?php

use App\Http\Controllers\ItemController;
use Illuminate\Support\Facades\Route;

Route::prefix('v1')->middleware('api.key')->group(function () {
    Route::get('/items', [ItemController::class, 'index']);
    Route::get('/items/{id}', [ItemController::class, 'show']);
    Route::post('/items/filter', [ItemController::class, 'filter']);
    Route::post('/items', [ItemController::class, 'store']);
});
```

---

Tambahkan di `.env`

```
IAE_API_KEY=NIM_KAMU
```

---

Cara Kerja:

```
Request masuk
     ↓
Middleware baca header X-IAE-KEY
     ↓
Cocok dengan IAE_API_KEY di .env?
     ↓              ↓
    YA              TIDAK
     ↓              ↓
Lanjut ke       Return 401
Controller      Unauthorized
```

**Cara Testing di Postman:**
| | Tanpa Header | Dengan Header |
| :--- | :--- | :--- |
| Header | - | `X-IAE-KEY: NIM_KAMU` |
| Response | 401 Unauthorized | 200 Success |

# AI RESPONSE

Berikut implementasi GraphQL menggunakan Lighthouse:

---

**Install Lighthouse & GraphQL Playground**

```
composer require nuwave/lighthouse
php artisan vendor:publish --tag=lighthouse-schema
composer require mll-lab/laravel-graphiql
```

---

`graphql/schema.graphql`

```
"A datetime string with format `Y-m-d H:i:s`"
scalar DateTime @scalar(class: "Nuwave\\Lighthouse\\Schema\\Types\\Scalars\\DateTime")

type Query {
    "Mengambil semua barang lelang (setara GET /api/v1/items)"
    items: [Item!]! @all

    "Mengambil detail barang berdasarkan ID (setara GET /api/v1/items/{id})"
    item(id: ID! @eq): Item @find
}

type Item {
    id: ID!
    name: String!
    description: String!
    starting_price: Int!
    current_highest_bid: Int!
    auction_status: String!
    auction_deadline: DateTime!
    image_url: String
    created_at: DateTime!
    updated_at: DateTime!
}
```

---

**Akses GraphQL Playground**

```
http://127.0.0.1:8000/graphiql
```

---

**Contoh Query di Playground**

**Setara** `GET /api/v1/items` **— Ambil semua barang:**

```
{
    items {
        id
        name
        description
        starting_price
        current_highest_bid
        auction_status
        auction_deadline
    }
}
```

**Setara** `GET /api/v1/items/{id}` **— Ambil detail barang:**

```
{
    item(id: 1) {
        id
        name
        description
        starting_price
        auction_status
    }
}
```

---

**Keunggulan GraphQL vs REST**
| **Aspek** | **REST API** | **GraphQL** |
| :--- | :--- | :--- |
| Pengambilan field | Semua field ikut terambil | Client pilih field yang dibutuhkan |
|Jumlah request | Bisa butuh beberapa endpoint | Cukup 1 query |
|Over-fetching | Sering terjadi | Tidak terjadi |
